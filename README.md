# 改造道 ｜ Kaizō-dō

> 青写真の、その先へ。── 技術と実装を“体で”理解し、生成AIで要件定義を速く・広く・深くする、無料・実践型の教材。

これは、後進の育成に悩む方への、私からの小さな贈り物です。

戦略を描き、To-Be を語り、ロードマップを引ける ── そういう優秀な上流人材は、すでにたくさん育っています。けれど「その構想を、実装に耐える要件まで生きたまま届ける」ところで、多くの人が静かに詰まります。CMS で何を持つのか、フロントで何を制御するのか、その粒度を誰がどう教えるのか ── 育てる側にとっても、ここは正解の見えにくい領域です。

生成AIの登場で、この景色は変わりました。コードを一行も書いたことのない人でも、ブラウザとチャットAIだけで、本物のブランドサイトを動かし、コンテンツの“型”を定義し、機能を実装し、結果を自分の目で確かめられる。技術が“見える”ようになることで、要件定義の解像度は確かに上がります。そして要件が鋭ければ、AI による実装はそのぶん滑らかになります。

この教材は、その体験を一本道で通せるように組んだものです。派手な主張はしません。育成に悩む誰かの手元で、静かに役に立てば、それで十分です。

## 誰のための教材か

育てる側にとっての「想定する受講者像」を、一人の人物として置いています。

> **高橋 亮介 / 32歳 / シニアコンサルタント**
> デジタル戦略・CX 領域。経営課題の整理、To-Be 像、施策ロードマップ、KPI ツリー、カスタマージャーニー、ステークホルダー資料 ── 上流の設計はすでに確かな武器。
> 一方で、CMS で何を管理すべきか、コンテンツタイプ／フィールド／テンプレートの違い、HTML/CSS/JavaScript が何をしているか、アクセシビリティや Core Web Vitals を要件としてどう書くか ── このあたりに、まだ手応えを持てずにいる。

この人物に向けて、教材はまず構想力を肯定し、そこを出発点として「実装まで届く要件」へと射程を広げていきます。最初からコードを書かせて離脱させない、という配慮も設計に含めています。

ドメインは **デジタルマーケティングの Web Content Management（WCM）** に絞っています。

## どう学ぶか ── 体験ループ

座学ではありません。各モジュールで、次の5ステップを一周します。

1. **依頼を受ける** ── 架空のステークホルダーから、現実的に少し曖昧な要望が届く
2. **AI と壁打ちして要件を定義する** ── ユーザーストーリー・受け入れ条件・コンテンツモデルの草案は AI に量産させ、判断（成功の定義／合意可能性／優先度）は自分が握る
3. **成果物をつくる** ── ストーリー＋受け入れ条件、コンテンツモデル、トレーサビリティ表などを実際に書き起こす
4. **AI に実装させる** ── 成果物を AI に渡し、生成されたコードを自分のサイトに反映する
5. **結果を体験し、振り返る** ── 動いたサイトを見て、「要件の質が結果をどれだけ動かしたか」を言葉にする

学習に必要なのは **ブラウザとチャット型の生成AI だけ**。ローカル環境もコマンドラインも要りません。GitHub の Web エディタで編集し、GitHub Pages で確認し、AI でコードを生成する ── すべてがブラウザの中で完結します。

各モジュールには、「**▷ AI に委ねる ／ ● 人が握る**」の線引きカード、コピーして使えるプロンプト、成果物のひな形を常設しています。

## 何を扱うか ── モジュール構成

舞台は、京都発の架空のプレミアム・スキンケア＆ウェルネスブランド **「凪 / NAGI」**（ブランドサイト。EC ではありません）。多言語・多地域、ガバナンス、複数のコンテンツタイプ ── WCM の論点が自然に必要になる世界です。学習順はモジュール番号のとおり（M0 → M8）。

## 二つの砂場 ── To-Be と As-Is

- `nagi/` ── リニューアル後の **NAGI（To-Be）**。学習者が改造していくブランドサイト。`nagi/data/*.json` がコンテンツの“型”と中身、表示層は素の JavaScript。
- `nagi-legacy/` ── **現行サイト（As-Is）**。M2 で診断する対象。ぱっと見はきれいでも、よく見ると不便が潜んでいます。

「コンテンツの**型**」と「**見た目**」が分離した状態を、自分の手で触って体感できる ── これがこの教材の中心の仕掛けです。型を雑に決めると、実装で壁にぶつかる。その痛みごと、設計図の上では決して見えなかったものが見えてきます。

## 技術メモ ── 「擬似ヘッドレス」

NAGI サイトは **構造化データ（JSON）＋クライアントサイド描画** で動く、ヘッドレス CMS のミニチュアです。ビルド工程はゼロ。GitHub の Web エディタで編集でき、GitHub Pages でそのまま表示されます。自分で小さなヘッドレスを建てた経験が、後半の方式判断（M7）を体感を伴って理解する土台になります。

なお、教材側（エディトリアル）と NAGI 側（プレミアム）で、あえて異なるデザインシステムを用意しています。ブランド表現の UI と、機能的な UI は別物 ── その対比自体も教材の一部です。

## ローカルでの確認

