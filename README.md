# 改造道 ｜ Kaizō-dō

> 青写真の、その先へ。── 技術と実装を“体で”理解し、生成AIで要件定義を速く・広く・深くする、無料・実践型の教材。

デジタルマーケティングの **Web Content Management（WCM）** 領域に絞った、ケーススタディ型のオンライン教材です。GitHub Pages 上で完結し、受講者は **ブラウザとチャット型の生成AI** だけで学習します。

## この教材が腹落ちさせること

**要件定義の質は、そのままプロダクトの質になる。そして生成AIは、技術が“見えている”人が使うほど、要件定義を速く・広く・深くする増幅器になる。**

## 構成（この縦切りリリース）

| パス | 内容 |
|---|---|
| `index.html` | 学習プラットフォームのトップ（主張・カリキュラム地図） |
| `course/m0.html` | **M0** ── オリエン＆ブリーフ。NAGIの世界観・ステークホルダー・プロジェクト憲章（T1） |
| `course/m4.html` | **M4** ── 機能要件（T9）。体験ループを1本フルで。「わざと失敗」フォーク付き |
| `nagi/` | 仮想ブランド **NAGI / 凪** のブランドサイト（学習者の「砂場」） |
| `nagi/data/*.json` | 構造化コンテンツ（＝「CMSの中身」）。`site` / `products` / `journal` |
| `assets/`, `nagi/assets/` | 2つの対比するデザインシステム（教材側＝エディトリアル／NAGI側＝プレミアム） |

学習導線：**M0 → NAGIサイト → M4** の一本道を完走できます。

## 技術メモ ── 「擬似ヘッドレス」

NAGI サイトは **構造化データ（JSON）＋クライアントサイド描画** で動く「ヘッドレスCMSのミニチュア」です。
ビルド工程ゼロ。GitHub の Web エディタで編集でき、GitHub Pages でそのまま表示されます。コンテンツの“型”と“見た目”が分離した状態を、受講者が自分の手で体験できます。

## ローカルでの確認

静的サイトです。任意の静的サーバで確認できます（`fetch` を使うため `file://` 直開きではなく HTTP 配信が必要）。

```sh
python3 -m http.server 8000
# → http://localhost:8000/
```

## 公開（GitHub Pages）

`.github/workflows/static.yml` がリポジトリ全体を Pages へデプロイします。`.nojekyll` により Jekyll 処理は無効化済み。
