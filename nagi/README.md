# NAGI / 凪 ── 学習用ブランドサイト（あなたの「砂場」）

これは改造道（Kaizō-dō）の教材で使う、**架空のプレミアム・ウェルネスブランド「凪 / NAGI」** のブランドサイトです。
受講者は、このサイトに「追加要件」を要件定義 → 実装していくことで、実践的に学びます。

## しくみ ──「擬似ヘッドレス」

ビルド工程はありません。各ページが `/data/*.json` を `fetch` して HTML に描画します。

```
nagi/
├── index.html         ホーム（hero / コレクション / リチュアル / ジャーナル）
├── products.html      コレクション一覧 ＋ ?id= で製品詳細
├── journal.html       ジャーナル一覧 ＋ ?id= で記事詳細
├── data/
│   ├── site.json      ブランド共通情報・ナビ・リチュアル
│   ├── products.json  製品（コンテンツタイプ：product）
│   └── journal.json   記事（コンテンツタイプ：journal_article）
└── assets/
    ├── css/nagi.css   デザインシステム
    └── js/nagi.js     レンダラー（表示層）
```

`data/*.json` が「CMSの中身」、`assets/js/nagi.js` が「表示層」。
**コンテンツの“型”と“見た目”が分離している**ことを、編集して体験してください。

## 砂場の始め方（ブラウザだけ）

1. このリポジトリを GitHub で **Fork**（または “Use this template”）。
2. リポジトリの **Settings → Pages** で公開を有効化。
3. 数分後、`https://<あなた>.github.io/<リポジトリ名>/nagi/` であなた専用のNAGIが見られます。
4. 編集は GitHub の **鉛筆アイコン（Webエディタ）** から。コミットすると Pages に反映されます。

## やってみよう（M4 の課題）

`course/m4.html`（M4 機能要件）の手順に沿って、新しいコンテンツタイプ **プレスリリース** を追加します。
`data/press.json` を作り、AIに `press.html` を生成させ、受け入れ条件で検収する ── 体験ループを1周まわします。
