M6 ── 非機能要件（T11）｜ 測る側

「速く・きれいに」を、
数値と基準に変える。

「速く・アクセシブルに・SEOに強く」── 形容詞のままでは、実装者は迷い、検収では揉めます。 品質の合格ラインは、数値（Core Web Vitals）と準拠基準（WCAG／JIS X 8341）で固定して初めて成立する。 M2で“言葉”で見つけた課題を、ここで“測れる要件”にします。

この回を終えると、「速い・見やすい・アクセシブル」を、数値と準拠基準で合意できるようになります。

① 依頼を受ける

M2 で課題が言語化され、リニューアルが本格化します。すると品質と統制の話で、二人のステークホルダーが動きます。

💬 デジタル・三宅 ＆ 法務・和田 からの依頼

三宅「品質の合格ラインを先に決めておきたい。速さもアクセシビリティもSEOも、『いい感じ』だと検収で揉めるので。」
和田「法務として、市場ごとに出していい表現を制御したい。日本でOKでも、欧州では言えない効能表現がある（R-04）。」

機能要件（M4）が「何ができるか」なら、非機能要件は「どの品質で・何を守って」。ここを形容詞で書くと、実装も検収も崩れます。M6のゴールは、品質を数値と準拠基準に、統制を承認フローに落とすことです。

② AIと壁打ちして要件を定義する

非機能はカテゴリが広く、抜けが致命傷になります。網羅はAIに委ね、合格ラインの数値と優先は人が握る。

▷ AIに委ねる

  • 非機能カテゴリのチェックリスト網羅
  • 各項目の指標と測り方の候補
  • 準拠基準（WCAG／JIS X 8341）の条文の要約
  • 見落としがちな非機能の洗い出し

● 人が握る

  • 合格ラインの数値（このサイトでLCP何秒か）
  • ビジネス影響に応じた優先度
  • ガバナンス（誰が・何を承認するか）
  • 「測れる言葉になっているか」の判定

握ったら、書く ── ここで決めた判断は、非機能要件一覧の数値・基準・測り方に書き留めて初めて「握った」ことになります。

🧭 コンサルの思考発散：非機能チェックリスト
あなたは経験豊富な非機能要件／Web品質のコンサルタントです。
多言語・多地域のブランドサイトの非機能要件を、次のカテゴリで漏れなくチェックリスト化してください：
性能(Core Web Vitals)／可用性・SLA／セキュリティ／アクセシビリティ(WCAG／JIS X 8341)／SEO／
運用・ガバナンス・承認フロー／i18n／拡張性／プライバシー(同意管理)。
各項目は「形容詞(速い等)」ではなく「指標・準拠基準・測り方」で書ける雛形を添えること。
最後に、ブランドサイトで見落とされがちな非機能を3つ指摘してください。

③ 形容詞を、数値と基準に変える

M6 の核心は翻訳です。同じ要望でも、形容詞のままか、測れる言葉にしたかで、実装も検収もまるで変わります。

✗ 形容詞のままだと

  • 「速いサイト」
  • 「アクセシブルに」
  • 「SEOに強く」
  • 「多言語対応」

合格か分からない。実装者は解釈で迷い、検収は主観で揉め、設計が止まる。

✓ 数値・基準にすると

  • LCP ≤ 2.5s／CLS ≤ 0.1／INP ≤ 200ms
  • WCAG 2.1 AA準拠・コントラスト ≥ 4.5:1
  • 全ページ固有の title・meta description・構造化データ
  • hreflang で言語別配信・タグIDは不変（M5と整合）

合格が一意に決まる。数値だから、誰が見ても同じ判定。検収できる。

● 出力は一例 数値は“このサイトの事情”で握るもの。AIの提案を、ビジネス影響と現実性で取捨します。プロンプトを自分で書けるようになるには →（Appendix）

✦ ここに、“測る”の力がある

「薄い気がする」は主観で、揉めます。「4.5:1 を下回る＝WCAG AA不合格」は事実で、合格ラインになる。非機能要件とは、品質を誰が見ても同じ判定にできる言葉にすることです。

✦ M2 の課題が、ここで“合格ライン”になる

M2 で見つけた「薄い本文」「CLS」「同一title」は、いま ≥4.5:1 / CLS≤0.1 / 固有title という検収可能な数値になりました。診断（M2）→ 測定・基準化（M6）は一本の線です。

⑤ 振り返り

形容詞は、検収できない。数値と準拠基準にして初めて、品質は合格ラインになる。
技術が見えると、「速く・きれいに」を測れる言葉に翻訳でき、実装者とも法務とも同じ土俵で話せる。
気づき 1

主観は揉め、数値は決まる

「薄い」は揉める。「4.5:1未満」は決まる。非機能は“誰が見ても同じ判定”にする技術。

気づき 2

ガバナンスも非機能

「市場別の表現規制」は、承認フローという非機能要件に落ちる。法務の懸念も、要件として握れる。

十分性チェック

セルフチェック合意の前に
  • □ 各要件が形容詞でなく数値・準拠基準になっているか
  • □ それぞれに測り方（指標・ツール）が紐づくか
  • □ 性能・a11y・SEO・i18n・ガバナンス・プライバシーに抜けがないか
  • □ ガバナンス（市場別の承認）を承認フローとして書けたか
  • □ M2 の課題が、合格ラインに翻訳されているか
